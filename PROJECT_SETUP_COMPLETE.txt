⚠️ CRITICAL SECURITY REMINDERS
═════════════════════════════════════════════════════════════════

NEVER DO THIS:
   ❌ Commit .env file to git
   ❌ Hardcode credentials in source code
   ❌ Share credentials in emails or chats
   ❌ Log credentials in debug output
   ❌ Commit API keys or tokens
   ❌ Use same credentials across environments

ALWAYS DO THIS:
   ✅ Copy from .env.example to .env
   ✅ Use environment variables for configuration
   ✅ Store credentials securely (AWS Secrets Manager, Azure Key Vault)
   ✅ Rotate credentials quarterly
   ✅ Review .gitignore before commits
   ✅ Verify credentials NOT in git: git log --all -S "password"
   ✅ Use separate credentials for dev/staging/prod

IF CREDENTIALS LEAK:
   1. Immediately rotate ALL credentials
   2. Notify security team
   3. Remove from git history
   4. Force push only with approval
